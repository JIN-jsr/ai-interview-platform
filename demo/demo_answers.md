# 演示回答通用模板

以下内容均为虚构演示材料，仅用于系统测试、课堂展示和比赛视频录制，不包含真实个人信息。

## 1. 自我介绍

您好，我目前主要准备计算机相关岗位，最近重点做过一个简历驱动的模拟面试平台。项目里我负责简历解析、岗位画像、知识点检索、面试流程和评分报告几个模块。我的优势是能把业务流程拆成稳定的工程闭环，同时比较重视异常处理和结果可解释性。

## 2. 项目介绍

这个项目的目标是根据候选人的简历和目标岗位生成模拟面试，并在回答后给出分析和评分报告。整体流程是先读取简历文本，提取技能、项目和岗位方向，再用知识库检索相关题目，最后结合 LLM 生成自然问题。如果 LLM 不可用，系统会切换到备用出题机制，保证演示和训练流程不中断。

## 3. 基础知识题

我回答基础知识题时会先说明概念，再补充适用场景和注意点。比如讲 Redis 缓存时，我会说明缓存可以降低数据库压力，但也要处理一致性、过期策略、热点 key 和降级方案。这样回答不只是背定义，也能体现工程使用中的风险意识。

## 4. 项目深挖题

如果追问项目难点，我会按背景、问题、方案和结果来回答。比如系统需要让问题既自然又有依据，我的做法是先由 RAG 决定知识点，再让 LLM 结合简历画像和历史回答组织表达，同时校验返回字段，保留 `knowledge_id`、参考答案和期望要点，避免生成不可追溯的问题。

## 5. 困难工程问题

遇到接口超时、模型返回异常或文件解析失败时，我会优先保证主流程可继续运行。具体做法包括设置超时时间、捕获异常、记录失败原因、使用本地规则兜底，并在界面上给出清楚提示。这样即使外部服务不稳定，用户仍然可以完成面试和报告生成。
