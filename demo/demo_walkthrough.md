# Demo Walkthrough

目标：在 6-8 分钟内展示完整闭环，突出“简历驱动、RAG 支撑、LLM 连续追问、评分反馈可解释”。

## 0:00-0:40 启动项目

```bash
.venv\Scripts\activate
streamlit run app.py
```

打开页面后，先展示顶部项目简介和侧边栏状态面板。

## 0:40-1:10 检查 LLM 状态

进入“项目说明与自检”页，查看 `USE_LLM`、`MODEL_NAME`、`LLM_BASE_URL`。点击“测试 LLM 连接”。

说明：如果 LLM 不可用，系统仍会使用本地备用机制完成闭环。

## 1:10-1:50 加载示例简历

在侧边栏选择“AI 应用开发示例”，点击“加载示例简历”。回到“简历输入与解析”页，确认文本已经填入。

## 1:50-2:40 生成用户画像

点击“解析简历并生成画像”。展示结构化简历、目标岗位、识别技能和面试重点。

## 2:40-3:20 展示 RAG 检索

进入“RAG 知识库”页，输入 `RAG LLM Embedding Chroma` 或 `后端 MySQL Redis API` 测试检索。展示知识点的 `category`、`difficulty`、`answer` 和 `follow_up`。

## 3:20-5:20 开始模拟面试

进入“模拟面试”页，点击“开始面试”。展示当前问题、题型、`generated_by`、难度、`knowledge_id`，并展开 `expected_points` 和 `reason`。

复制 `demo/sample_answers_ai_app.md` 中 3-4 个回答进行演示。强调系统会记录回答、分析覆盖率，并继续生成下一题或追问。

## 5:20-6:30 查看记录与分析

进入“面试记录与分析”页，展示每题的问题、用户回答、回答分析、参考答案和下载 JSON。

## 6:30-7:30 生成评分报告

进入“评分报告”页，点击“生成正式评分报告”。展示总分、等级、五维度评分、评分依据、优势、主要问题和建议。

## 7:30-8:00 下载报告并收尾

点击下载 JSON 或 Markdown 报告。最后总结系统闭环：

```text
简历输入 -> 用户画像 -> RAG 检索 -> 连续面试 -> 回答分析 -> 评分报告
```
