# RAG 知识库构建说明

## 知识库位置

```text
data/knowledge_base.json
```

## 知识库规模

当前知识库包含 80 条有效知识条目，覆盖：

- Python
- Java
- 数据结构与算法
- 数据库与缓存
- 计算机网络
- 操作系统
- 软件工程与 AI 应用

## 条目格式

每条知识包含：

```json
{
  "id": "db_001",
  "category": "数据库与缓存",
  "tags": ["MySQL", "索引", "数据库"],
  "difficulty": "medium",
  "question": "数据库索引的作用是什么？",
  "answer": "索引可以加快数据查询速度，但会占用额外存储空间，并降低部分写入、更新和删除操作的效率。",
  "follow_up": ["什么情况下索引可能失效？", "B+ 树索引有什么特点？"],
  "source": "课程资料、官方文档与公开学习资料整理"
}
```

## 检索逻辑

当前版本使用可解释的本地关键词检索：

```text
简历解析结果 / 用户画像
↓
提取目标岗位、技能、项目关键词
↓
匹配知识库 category、tags、question、answer
↓
计算匹配分数
↓
类别多样化筛选
↓
返回 Top-K 基础知识问题
```

## 为什么不用复杂向量库

本项目只有一周开发周期。为了保证演示稳定性，当前采用本地关键词检索。优点是：

- 不依赖额外向量数据库
- 不依赖 embedding API
- 速度快
- 可解释性强
- 评委容易看到知识库和检索依据

后续可扩展为 FAISS / Chroma 向量检索。
