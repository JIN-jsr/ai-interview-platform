# 演示视频脚本

建议视频控制在 7 分钟左右，最长不超过 8 分钟。演示材料必须使用 `demo/` 中的虚构简历和回答模板，不展示真实 API Key、真实简历或真实面试记录。

## 录制前检查

- [ ] 使用虚构示例简历。
- [ ] 隐藏 `.env` 和 API Key。
- [ ] 关闭无关浏览器标签页和系统通知。
- [ ] 提前测试 LLM 连接。
- [ ] 准备 `USE_LLM=false` fallback 方案。
- [ ] 准备至少一份已生成报告的历史记录，用于能力成长曲线展示。
- [ ] 检查麦克风和录屏区域。
- [ ] 避免展示本地个人路径和私人文件。

## 0:00-0:30 项目定位、首页与目标

展示首页，说明这是一个面向计算机相关学生的 AI 模拟面试与能力提升平台，核心目标是从简历出发，完成面试训练、回答分析和评分复盘。

## 0:30-1:00 本地运行与演示模式

展示启动方式：

```bash
.venv\Scripts\activate
streamlit run app.py
```

说明也可在完成首次环境配置后双击 `start_app.bat`。在侧边栏打开“演示模式”，选择一个虚构岗位样例。

## 1:00-1:50 简历输入、结构化画像与岗位匹配提醒

加载示例简历，点击“解析简历并生成画像”。展示：

- TXT / PDF / DOCX 与多文件材料支持。
- 候选人画像。
- 面试重点与潜在薄弱项。
- 简历方向判断、岗位匹配提醒和简历优化建议。

## 1:50-3:40 面试题、RAG 依据与动态追问

开始模拟面试，展示 2-3 道题即可。重点说明：

- 问题由 RAG 条目和候选人画像驱动。
- LLM 成功时生成自然中文问题。
- 题目保留生成方式、难度、知识点、期望要点和参考依据。
- 回答后系统会继续追问或切换到下一题。

不要把内部 ID 或原始 JSON 作为展示重点。

## 3:40-4:10 回答分析、修改并重新提交

进入面试记录与分析页，展示回答覆盖率、关键词、缺失要点和临时评分。演示“修改并重新提交最近回答”，说明系统会回滚对应记录并重新生成后续问题。

## 4:10-5:30 五维度报告与辅助诊断

进入评分报告页，生成正式评分报告。展示：

- 总分和等级。
- 五维度评分。
- 问题难度与类型分布。
- 回答稳定性。
- 岗位能力覆盖。
- 薄弱点卡片。
- 简历优化建议和学习建议。

说明分数由本地规则计算，LLM 只润色文字，不改变权重或分数。

## 5:30-5:55 多格式导出

展示下载按钮：

- Markdown
- JSON
- 完整报告长图 PNG
- 摘要海报 PNG

说明长图适合归档，摘要海报适合展示。

## 5:55-6:30 历史记录与能力成长曲线

切换历史面试记录，进入能力成长曲线页面。展示总分趋势、五维度变化和后续提升建议。

## 6:30-7:00 fallback、隐私与总结

说明系统在 LLM 不可用、网络超时或 `USE_LLM=false` 时会启用本地规则 fallback。强调 `.env` 不上传、真实简历不提交、演示数据均为虚构。最后总结完整闭环：

```text
简历输入 -> 画像生成 -> RAG 检索 -> 连续面试 -> 回答分析 -> 评分报告 -> 成长复盘
```
