# 演示视频脚本

最终演示视频约 7 分 55 秒，必须严格控制在 8 分钟以内。正式提交前应再次核查导出文件的实际时长，避免因片尾、淡出或重新编码超过限制。

本脚本文档用于说明最终演示视频的结构、重点和呈现边界，不逐字复现全部旁白。演示视频已包含完整旁白、字幕和背景音乐，整体使用虚构简历、虚构候选人回答和虚构历史面试记录，不展示 API Key、私人简历或个人面试数据。

音乐前半段使用 `Sappheiros — Lights`，约在面试结束并点击 `查看报告` 的 4:26 附近，通过柔和交叉淡入淡出转入 `Sappheiros — Embrace`。片尾保留简短音乐署名。片尾授权信息应以实际下载来源为准，预期展示格式类似：

```text
Music:
“Lights” — Sappheiros
“Embrace” — Sappheiros
Licensed under CC BY 3.0
```

视频画面不使用常规右上角高亮说明条，仅保留章节标题、旁白字幕、应用自身界面文字；DeepSeek 首次出现时可使用一条短澄清遮罩，避免误解其在系统中的角色。

## 0:48-1:20 面试配置、演示模式与实时状态

展示新建面试、目标岗位、难度选择、答后即时分析开关、LLM 辅助反馈润色开关、实时面试状态和演示模式。

## 1:20-1:58 简历解析与候选人画像

展示虚构后端开发简历、一键加载、压缩后的解析等待、岗位匹配提醒、匹配分析、简历优化建议、基本摘要、结构化解析结果、候选人画像、面试重点和 RAG 相关基础问题。

这一段说明：一次简历解析结果会被后续多个环节复用，包括 RAG 检索、问题生成、项目深挖、岗位匹配建议和最终报告建议。系统不是随机抽题，而是先形成候选人画像，再围绕岗位要求组织面试。

## 1:58-4:26 连续面试主流程

这是视频的主交互段，展示从开始面试到结束并点击 `查看报告` 的完整过程。画面只保留代表性能力，不逐个描述复制粘贴动作。

重点展示：

- 开始模拟面试。
- 多轮连续问题。
- RAG 驱动的问题生成。
- 预期回答要点。
- 问题生成依据。
- RAG 出题依据和证据。
- 项目相关深挖。
- 上下文追问。
- 实时面试状态。
- 每题提交后的答后即时分析。
- 详细分析展开。
- 可选的 LLM 辅助反馈润色。
- 修改并重新提交最近一次回答。
- 分析结果更新，且不产生重复回答记录。
- 完成本轮面试。
- `导出即时分析`。
- 打开 Markdown 复盘文件。
- 约 4:26 点击 `查看报告`。

演示中的候选人回答由 DeepSeek 辅助生成，仅作为虚构测试输入。DeepSeek 不属于平台正式运行流程；平台本身负责出题、分析、评分与报告生成。项目实际配置的 LLM 为 `qwen3.7-plus`。

反馈边界需要讲清楚：本地规则负责覆盖点、缺失点、风险提示和评分证据；`qwen3.7-plus` 在启用时只辅助润色反馈表达，不修改评分数据或五维权重。

即时分析 Markdown 导出用于逐题复盘，包含问题、用户回答、即时分析、覆盖点、缺失点、风险表述和改进建议，不包含完整参考答案。

## 4:26 后：总体分析与综合报告阶段

约 4:26 是视频叙事的分界点：从“正在进行的面试训练”转入“整轮表现复盘与综合评价”。音乐也在此处由 `Lights` 柔和切换到 `Embrace`，形成更沉稳的报告阶段氛围。

## 总体回答分析

展示面试记录与回答分析页中的总体回答分析模块，包括：

- 整体表现摘要。
- 跨题稳定优势。
- 重复遗漏。
- 反复出现的风险表述或认知误区。
- 优先改进方向。
- 选定题目摘要。
- 本轮面试记录 JSON 导出。

单题即时分析解决“这一题应该怎么改”；总体回答分析跨题识别稳定优势、重复问题和训练优先级，不重新修改最终五维评分。

## 正式五维评分报告

展示正式评分报告页，重点包括：

- 总分和等级。
- 五维雷达图。
- 问题难度与类型分布。
- 回答稳定性。
- 岗位能力覆盖。
- 维度详情。
- 评分证据。
- 优势总结。
- 主要问题。
- 改进建议。
- 简历与岗位匹配建议。
- 薄弱点卡片。
- 错题与薄弱知识总结。
- 推荐学习计划。

固定评分维度和权重为：

| 维度 | 权重 |
|---|---:|
| 基础知识掌握程度 | 25% |
| 项目理解深度 | 25% |
| 回答逻辑性 | 20% |
| 表达完整性 | 15% |
| 岗位匹配度 | 15% |

分数由本地规则证据生成，LLM 不直接改变分数或权重。报告用于训练、自我复盘和面试准备，不作为真实招聘决策依据。

## 多格式导出

展示评分报告导出能力：

- 评分报告 JSON：适合保留结构化评分数据。
- 评分报告 Markdown：适合阅读、归档和后续修改。
- 完整报告长图 PNG：适合保存完整页面证据。
- 摘要海报 PNG：适合展示关键结论。

这一段只说明不同格式的用途，不需要逐个展示复杂鼠标动作。

## 历史记录与能力成长

展示多份虚构历史面试记录，包括记录切换、重命名记录、总分趋势、五维能力趋势、成长摘要、维度变化分析和后续提升建议。

这一段说明系统从一次性评分扩展到长期训练复盘。用户可以比较多轮表现，识别持续进步的能力和反复出现的短板。

## 运行检查、隐私与 fallback

展示 LLM 连接测试成功、项目文件状态、提交前检查、推荐命令、文档位置、实时模型状态和备用问题生成机制。

说明边界：

- `.env` 不提交。
- 真实 API Key 不展示。
- 演示使用虚构简历、虚构回答和虚构历史记录。
- 如果 LLM 被关闭、超时或返回无效结果，本地规则 fallback 会保留基础流程。
- 规则 fallback 出题用于保证流程稳定，不改变“训练演示项目”的定位。
- 当前项目不宣称云部署，也不宣称生产级安全合规能力。

## 系统流程动画

最后用流程动画回顾系统闭环：

```text
简历输入 / 文件上传
→ 简历文本读取
→ 结构化简历解析
→ 候选人画像与岗位匹配分析
→ RAG 知识库检索
→ LLM + RAG 协同出题
→ 规则 fallback 出题
→ 模拟面试对话
→ 回答分析
→ 五维度评分报告
→ 报告下载 / 历史记录 / 成长曲线
```

fallback 分支最终回到主面试工作流，用于说明系统在网络或模型异常时仍能完成基础训练闭环。
