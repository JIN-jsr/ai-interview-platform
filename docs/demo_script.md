# 演示视频脚本

建议视频控制在约 7 分 20 秒，最长不超过 8 分钟。演示材料必须使用 `demo/` 中的虚构简历和回答模板，不展示真实 API Key、真实简历或真实面试记录。

## 录制前检查

- [ ] 使用虚构示例简历。
- [ ] 隐藏 `.env` 和 API Key。
- [ ] 关闭无关浏览器标签页和系统通知。
- [ ] 提前测试 LLM 连接。
- [ ] 准备 `USE_LLM=false` fallback 方案。
- [ ] 准备至少一份已生成报告的历史记录，用于能力成长曲线展示。
- [ ] 检查麦克风和录屏区域。
- [ ] 避免展示本地个人路径和私人文件。

## 0:00-0:30 项目定位、首页与目标

展示首页，说明这是一个面向计算机相关学生的 AI 模拟面试与能力提升平台，核心目标是从简历出发，完成面试训练、回答分析和评分复盘。

## 0:30-1:00 本地运行与演示模式

展示启动方式：

```powershell
.venv\Scripts\activate
streamlit run app.py
```

说明也可在完成首次环境配置后双击 `start_app.bat`。在侧边栏打开“演示模式”，选择一个虚构岗位样例。

## 1:00-1:50 简历输入、结构化画像与岗位匹配提醒

加载示例简历，点击“解析简历并生成画像”。展示：

- TXT / PDF / DOCX 与多文件材料支持。
- 候选人画像。
- 面试重点与潜在薄弱项。
- 简历方向判断、岗位匹配提醒和简历优化建议。

## 1:50-3:40 面试题、RAG 依据与动态追问

开始模拟面试，展示 2-3 道题即可。重点说明：

- 问题由 RAG 条目和候选人画像驱动。
- LLM 成功时生成自然中文问题。
- 题目保留生成方式、难度、知识点、期望要点和参考依据。
- 回答后系统会继续追问或切换到下一题。
- 完整面试会尽量覆盖项目深挖和上下文追问。

不要把内部 ID 或原始 JSON 作为展示重点。

## 3:40-4:35 回答分析、修改并重新提交

在模拟面试页提交一段准备好的回答，展示“本题回答分析”卡片。按照“提交回答 -> 即时分析出现 -> 展开详细分析 -> 展示 LLM 辅助润色边界 -> 修改并重新提交 -> 分析结果更新”的顺序演示。简要指出“回答亮点”“建议补充”和风险提示，展开“查看详细分析”停留 2-3 秒，再演示“修改回答 / 重新发送”。

推荐讲解：

```text
每次提交回答后，系统都会生成轻量即时分析，指出已经覆盖的要点、最重要的遗漏和下一步改进方向。详细依据可以按需展开，也可以修改最近一次回答重新提交。即时反馈用于训练，最终五维评分仍在完整面试结束后统一生成。
```

如果演示环境 LLM 可用，可顺带说明“LLM 辅助反馈润色”只优化表达，不改变本地规则评分；如果 LLM 不可用，系统会自动显示本地规则反馈。

完整回答后，展示结束提示处的“导出即时分析”和“查看报告”两个并排按钮。点击或说明“导出即时分析”会生成 Markdown 复盘文件，包含每题问题、回答、覆盖点、缺失点、风险提示和改进建议，但不包含完整参考答案；随后再进入正式五维评分报告，明确区分逐题训练反馈和整轮综合评分。

## 4:35-5:55 五维度报告与辅助诊断

进入评分报告页，生成正式评分报告。展示：

- 总分和等级。
- 五维度评分。
- 问题难度与类型分布。
- 回答稳定性。
- 岗位能力覆盖。
- 薄弱点卡片。
- 评分置信度和项目证据充分性提示。
- 简历优化建议和学习建议。

说明分数由本地规则计算，LLM 只润色文字，不改变权重或分数。评分是训练型辅助评价，不作为真实招聘录用依据。

## 5:55-6:20 多格式导出

展示下载按钮：

- Markdown
- JSON
- 完整报告长图 PNG
- 摘要海报 PNG

说明长图适合归档，摘要海报适合展示。

## 6:20-6:55 历史记录与能力成长曲线

切换历史面试记录，进入能力成长曲线页面。展示总分趋势、五维度变化和后续提升建议。

## 6:55-7:20 fallback、隐私与总结

说明系统在 LLM 不可用、网络超时或 `USE_LLM=false` 时会启用本地规则 fallback。强调 `.env` 不上传、真实简历不提交、演示数据均为虚构。最后总结完整闭环：

```text
简历输入 -> 画像生成 -> RAG 检索 -> 连续面试 -> 即时分析 -> 逐题复盘导出 -> 五维评分报告 -> 成长复盘
```
