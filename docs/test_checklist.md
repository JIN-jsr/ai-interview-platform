# 测试清单

本清单用于提交前人工测试。不要预先勾选未实际验证的项目。

## 1. 安装与全新复现

- [ ] 从 GitHub 全新下载 ZIP 或重新 clone。
- [ ] 创建全新虚拟环境。
- [ ] 安装 `requirements.txt` 成功。
- [ ] 不配置 `.env` 也能启动页面。
- [ ] 运行 `python scripts/self_check.py` 无 ERROR。

## 2. 启动

- [ ] `streamlit run app.py` 可以启动。
- [ ] 双击 `start_app.bat` 可以启动。
- [ ] 浏览器能打开本地页面。
- [ ] 首页标题和浏览器标签页为中文。

## 3. 简历输入

- [ ] 手动粘贴简历可以解析。
- [ ] TXT 简历可以读取。
- [ ] PDF 简历可以读取。
- [ ] DOCX 简历可以读取。
- [ ] 多文件合并输入可用。
- [ ] 上传文件可撤回。

## 4. 演示模式

- [ ] 后端开发示例可加载。
- [ ] 前端开发示例可加载。
- [ ] AI 应用开发示例可加载。
- [ ] 数据分析示例可加载。
- [ ] 软件测试示例可加载。
- [ ] 对应岗位回答模板可显示。

## 5. 简历解析与岗位匹配

- [ ] 可以生成结构化简历。
- [ ] 可以生成候选人画像。
- [ ] 可以显示面试重点和潜在薄弱项。
- [ ] 目标岗位选择生效。
- [ ] 岗位匹配提醒生效。
- [ ] 简历优化建议生效。

## 6. RAG

- [ ] RAG 知识库页面可以检索。
- [ ] 解析简历后能推荐 RAG 问题。
- [ ] RAG 出题依据可以展示。
- [ ] 期望回答要点可以展示。
- [ ] 参考依据和项目场景可以展示。

## 7. LLM 面试

- [ ] 配置有效 API 后 `USE_LLM=true` 可用。
- [ ] LLM 出题时 `generated_by=llm`。
- [ ] LLM 题目保留 `knowledge_id`、`expected_points`、`reference_answer`。
- [ ] LLM 失败时不会中断面试。

## 8. fallback

- [ ] `USE_LLM=false` 时可完整运行。
- [ ] fallback 出题时 `generated_by=rule_fallback`。
- [ ] fallback 原因只在备用出题机制下显示。

## 9. 面试过程与回答编辑

- [ ] 可以开始面试。
- [ ] 可以连续回答多轮。
- [ ] 项目追问可触发。
- [ ] 基础知识追问可触发。
- [ ] 修改并重新提交最近回答可用。
- [ ] 完整回答 8 题后输入框消失。

## 10. 评分报告

- [ ] 完整报告可以生成。
- [ ] 未完成面试报告有阶段性提醒。
- [ ] 总分、等级、五维度评分显示正常。
- [ ] 回答稳定性显示正常。
- [ ] 问题类型分布显示正常。
- [ ] 岗位能力覆盖显示正常。
- [ ] 薄弱点卡片显示正常。

## 11. 导出

- [ ] Markdown 下载可用。
- [ ] JSON 下载可用。
- [ ] 完整长图 PNG 可用。
- [ ] 摘要海报 PNG 可用。
- [ ] 中文字体正常。
- [ ] 长文本无明显重叠或截断异常。

## 12. 历史记录与成长曲线

- [ ] 本地历史会话保存成功。
