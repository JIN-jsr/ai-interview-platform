# 项目设计文档初稿

## 1. 项目背景与目标

本项目面向计算机相关专业学生，开发一款 AI 模拟面试与能力提升软件。系统支持用户输入或上传简历，自动提取技术能力、项目经历和岗位方向，并基于 RAG 知识库和连续对话机制开展模拟技术面试。面试结束后，系统从基础知识、项目理解、回答逻辑、表达完整性和岗位匹配度五个维度生成评分报告。

## 2. 系统功能概述

系统主要包含以下功能：

1. 简历输入与解析
2. 用户画像与面试重点生成
3. RAG 知识库检索
4. 连续文字面试
5. 项目经历深挖
6. 基于回答内容的追问
7. 面试记录与回答分析
8. 五维度评分反馈报告

## 3. 技术栈

- Python：核心开发语言
- Streamlit：前端界面与交互展示
- JSON：知识库、解析结果和报告存储
- pdfplumber：PDF 简历文本读取
- python-docx：DOCX 简历文本读取
- requests：调用 OpenAI-compatible LLM API
- python-dotenv：本地 API Key 配置管理

## 5. 简历解析设计

系统支持 TXT、PDF、DOCX 和直接文本输入。若 LLM 已配置，系统调用大模型将简历解析为结构化 JSON；若未配置或调用失败，则自动使用本地规则解析，保证演示稳定性。

## 6. RAG 知识库设计

知识库包含 80 条计算机基础知识，覆盖 Python、Java、数据结构与算法、数据库、网络、操作系统、软件工程和 AI 应用等方向。系统根据简历识别出的技能和目标岗位，从知识库中检索相关问题，并将其用于基础知识考察和追问。

## 7. 面试流程设计

面试流程分为：

1. 自我介绍
2. 项目经历介绍
3. 项目难点深挖
4. RAG 基础知识考察
5. 基于回答内容的追问
6. 岗位匹配综合问题
7. 评分反馈

## 8. 回答分析设计

每轮回答后，系统记录问题类型、知识点 ID、用户回答和参考答案，并分析：

- 回答长度
- 技术关键词
- 参考答案覆盖度
- 逻辑表达线索
- 临时评分
- 存在问题
- 改进建议

## 9. 评分反馈设计

评分维度与赛题要求一致：

| 维度 | 权重 |
|---|---:|
| 基础知识掌握程度 | 25% |
| 项目理解深度 | 25% |
| 回答逻辑性 | 20% |
| 表达完整性 | 15% |
| 岗位匹配度 | 15% |

最终报告包含总分、等级、维度分数、评分依据、主要问题和后续提升建议。

## 10. API Key 安全处理

项目使用 `.env` 保存真实 API Key，并通过 `.gitignore` 排除该文件。仓库仅提供 `.env.example` 作为配置模板，避免密钥泄露。
